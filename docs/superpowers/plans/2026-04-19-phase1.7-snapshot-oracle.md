# Phase 1.7 — Snapshot oracle: sift → agx publishing surface

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal.** Ship the sift-side primitives that turn sift's content-addressed snapshot store into a queryable substrate any consumer (agx, eval scripts, third-party tools) can address. Three new commands, one schema, one suite-conventions update. Zero agx changes.

**Reframe pitch (from the conversation that produced this plan).**

> **sift** = per-turn snapshot oracle for the agent's file world
> **agx** = navigator that addresses it (timeline + corpus + diff)
> **git** = coarse approval signal that closes the loop

Three layers, independent codebases, one-way coupling per direction. This phase ships the sift half of the publish-and-consume contract that turns the framing from a slogan into a usable substrate.

**Further reframe (added later same day): agent-as-user.** The
primary user of sift is the agent (Claude, Gemini, Cline, …), not
the human. The human's only direct sift touchpoint is `git commit`
(which auto-accepts via the optional post-commit hook installed by
`sift init`). Querying history, reverting a file, comparing two
turns — these happen conversationally: the user asks the assistant,
the assistant runs the appropriate sift command. The `sift review`
TUI is a power-user escape hatch.

Implications baked into this plan:

- **JSON output is the primary contract.** Every new command in this
  phase ships `--format json` at first-class parity with text output.
  The agent parses JSON and renders a brief summary back to the user.
- **Stable schemas.** `sift_export_version` integer on the export;
  `path → SHA-1-hex` shape on `sift state` output, sorted via
  `BTreeMap` for deterministic diffs.
- **Agent guide ships alongside this plan.** `docs/agent-guide.md`
  documents the command cookbook so the agent knows when to reach
  for which command. `sift init` (Phase 1.7.3) optionally writes a
  sift-aware section into `CLAUDE.md` so the agent picks it up
  without per-session user briefing.
- **The Phase 1.4 validation gate shifts.** The signal is now
  "does the agent reflexively reach for `sift state` /
  `sift log --path` / `sift undo <id>` when the user asks about
  history?" — not "does the human press `t` in the TUI?" The
  agent's behavior is visible in any conversation transcript,
  which is more falsifiable.

**Scope boundary.** No agx code changes. No new runtime deps. No automatic git hooks (sift listens to git, never inserts). All three subplans are sift-side and git-side only — orthogonal to whether Phase 1.4 dogfood validates the `t` keybind.

**Philosophy check.** All three primitives expose granular substrate to human pattern recognition; none of them automate the recognition. `sift state` returns a hash map — what to look at, not a verdict. `sift export` ships JSON — for the user (or downstream tool) to read. `sift accept --by-commit` only accepts entries the user already endorsed via `git commit`; it does not infer endorsement. Each fits guiding principle #8 ("human judgment is the point; automation of recognition is a regression").

**Tech stack.** Rust, serde_json (already in-tree), `std::process::Command` for the git shell-out. No new crates.

- [ ] **Step 5: Implement**

  ```rust
  pub fn reconstruct_state_at_turn(
      ledger: &[LedgerEntry],
      at_turn: u32,
  ) -> BTreeMap<PathBuf, String> {
      reconstruct_state_at_turn_with_options(ledger, at_turn, IncludeReverted::No)
  }

  pub fn reconstruct_state_at_turn_with_options(
      ledger: &[LedgerEntry],
      at_turn: u32,
      include_reverted: IncludeReverted,
  ) -> BTreeMap<PathBuf, String> {
      // BTreeMap for deterministic JSON output ordering.
      let mut map = BTreeMap::new();
      // Sort by turn ascending so latest write wins via insert-overwrite.
      let mut sorted: Vec<&LedgerEntry> = ledger.iter().filter(|e| e.turn <= at_turn).collect();
      sorted.sort_by_key(|e| e.turn);
      for e in sorted {
          if e.status == Status::Reverted && include_reverted == IncludeReverted::No {
              continue;
          }
          if let Some(hash) = &e.snapshot_after {
              map.insert(e.path.clone(), hash.clone());
          } else {
              // Delete op: drop the path from the world.
              map.remove(&e.path);
          }
      }
      map
  }
  ```

- [ ] **Step 6: Add `--baseline` variant**

  Returns `path → snapshot_before` for each path's first appearance. Useful as a comparison endpoint for diffs (state-at-turn-N vs baseline = "what the agent created or changed cumulatively").

  ```rust
  pub fn reconstruct_baseline(ledger: &[LedgerEntry]) -> BTreeMap<PathBuf, Option<String>> {
      let mut sorted: Vec<&LedgerEntry> = ledger.iter().collect();
      sorted.sort_by_key(|e| e.turn);
      let mut map = BTreeMap::new();
      for e in sorted {
          // First-seen-wins.
          map.entry(e.path.clone()).or_insert_with(|| e.snapshot_before.clone());
      }
      map
  }
  ```

  ```rust
  pub const EXPORT_SCHEMA_VERSION: u32 = 1;

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct SiftExport {
      pub sift_export_version: u32,
      pub session_id: String,
      pub started_at: DateTime<Utc>,
      pub ended_at: Option<DateTime<Utc>>,
      pub transcript_path: Option<PathBuf>,
      pub turns: Vec<TurnExport>,
  }

  #[derive(Debug, Clone, Serialize, Deserialize)]
  pub struct TurnExport {
      pub turn: u32,
      pub entries: Vec<LedgerEntry>,
  }

  pub fn build(meta: &SessionMeta, ledger: Vec<LedgerEntry>) -> SiftExport {
      // Group by turn; ledger may already be sorted but don't assume.
      let mut by_turn: BTreeMap<u32, Vec<LedgerEntry>> = BTreeMap::new();
      for e in ledger {
          by_turn.entry(e.turn).or_default().push(e);
      }
      let turns = by_turn
          .into_iter()
          .map(|(turn, entries)| TurnExport { turn, entries })
          .collect();
      SiftExport {
          sift_export_version: EXPORT_SCHEMA_VERSION,
          session_id: meta.id.clone(),
          started_at: meta.started_at,
          ended_at: meta.ended_at,
          transcript_path: meta.transcript_path.clone(),
          turns,
      }
  }
  ```

- [ ] **Step 2: Implement**

  ```rust
  fn paths_in_commit(cwd: &Path, git_ref: &str) -> Result<Vec<PathBuf>> {
      let output = Command::new("git")
          .args(["diff", "--name-only", &format!("{git_ref}~..{git_ref}")])
          .current_dir(cwd)
          .output()
          .with_context(|| format!("running git diff for {git_ref}"))?;
      anyhow::ensure!(
          output.status.success(),
          "git diff exited {}: {}",
          output.status,
          String::from_utf8_lossy(&output.stderr)
      );
      let stdout = String::from_utf8_lossy(&output.stdout);
      let paths = stdout
          .lines()
          .filter(|l| !l.trim().is_empty())
          .map(PathBuf::from)
          .collect();
      Ok(paths)
  }
  ```

  ```rust
  pub fn run_by_commit(cwd: &Path, git_ref: &str, apply: bool) -> Result<()> {
      let paths = paths_in_commit(cwd, git_ref)?;
      let path_set: HashSet<PathBuf> = paths.into_iter().collect();
      let session = Session::open_current(Paths::new(cwd))?;
      let store = Store::new(&session.dir);
      let pending = store.list_pending()?;

      let mut would_accept = Vec::new();
      let mut diverged = Vec::new();
      let mut ignored_paths = path_set.clone();

      for entry in &pending {
          if !path_set.contains(&entry.path) {
              continue;
          }
          ignored_paths.remove(&entry.path);
          let on_disk_hash = sha1_of_file(&cwd.join(&entry.path))?;
          if Some(&on_disk_hash) == entry.snapshot_after.as_ref() {
              would_accept.push(entry.id.clone());
          } else {
              diverged.push((entry.id.clone(), entry.path.clone()));
          }
      }

      // Report.
      println!("sift accept --by-commit {git_ref}:");
      println!("  {} entries match (post-state == committed content)", would_accept.len());
      println!("  {} entries diverged (file changed since the agent wrote it)", diverged.len());
      println!("  {} commit paths had no matching pending entry (user edits, ignored)", ignored_paths.len());

      if !apply {
          println!("  (dry-run — pass --apply to finalize)");
          return Ok(());
      }
      for id in &would_accept {
          store.finalize(id, Status::Accepted)?;
      }
      println!("  accepted {}", would_accept.len());
      Ok(())
  }
  ```

## Acceptance criteria

The full Phase 1.7 ships when:

1. **`sift state`** — On a 10-turn session, `sift state --at-turn 5 > a.json` and `sift state --at-turn 8 > b.json` followed by `diff <(jq -S . a.json) <(jq -S . b.json)` shows exactly which paths differ between the two turns. `--include-reverted` toggles whether reverted writes participate; `--baseline` returns the pre-state map.

2. **`sift export --format json`** — Output is parseable, validates against `docs/export-schema.md`, and `sift_export_version` is the integer 1. `serde_json::from_str(...)` round-trips losslessly.

3. **`sift accept --by-commit HEAD`** — After `git commit`, run accepts every pending entry whose post-state hash matches the committed file's content; leaves diverged entries pending with a clear hint; ignores commit paths that have no matching pending entry. `--dry-run` is the default; `--apply` finalizes.

4. **Suite-conventions doc** — §5 sift section flipped from "planned" to "shipped"; §10 sift retrofit row closed; the agx-overlay row stays open as a downstream-consumer marker.

5. **README** — Lead pitch, "Pairs well with" agx bullet, and the usage block reflect the shipped state.

6. **All workspace tests pass** under `cargo test --workspace`. Clippy clean under `-D warnings`.

7. **Trial check** — The end-to-end "iterate on a prompt" loop a developer would actually run completes without redundant approval steps:

   ```
   # baseline run
   <agent runs N turns>
   git add . && git commit -m "baseline"
   sift accept --by-commit HEAD --apply

   # tweak prompt, re-run
   <agent runs from same baseline>
   sift state --at-turn N --json > current.json
   sift state --at-turn 0 --json > baseline.json
   diff <(jq -S . baseline.json) <(jq -S . current.json) | less
   ```

   If this flow has any "approve twice" or "scroll the transcript to remember what changed" friction, the phase failed its trial test (per stepwise principle #8).

## Validation gate

The Phase 1.4 dogfood (separate from this phase) should now ask one extra falsifiable question alongside the `t`-keybind one:

| Signal | Verdict |
|--------|---------|
| Maintainer reaches for `sift state --at-turn N` reflexively when comparing two points in a conversation; the diff is precise enough that questions like "what changed in turn 7?" get a one-line answer | Reframe validated. The snapshot-oracle pitch holds. |
| `sift state` is used occasionally but `sift list --turn N` would have sufficed | Reframe partial. Keep `sift state` (cheap to maintain), de-emphasize the pitch. |
| `sift state` is never used; `git diff` + scrolling is good enough | Reframe falsified. Drop the snapshot-oracle pitch from README and revert to "writable review gate". |

Write the verdict into the same `HANDOFF.md` as the Phase 1.4 verdict.

## Out of scope for this plan

- **Agx overlay rendering** (decorating timeline steps with sift status, time-travel diff between two agx-selected turns). Tracked in agx's own roadmap as a downstream consumer.
- **`sift export --format md / patch / bundle`** — Phase 4.3 work. Phase 1.7 only ships the `json` format.
- **Cross-session diff convenience wrappers** (the old Phase 4.5 use case). Once `sift state` ships, this is a one-liner shell pipeline (`sift state --session A | sift state --session B | diff`); a dedicated subcommand can land later if dogfood surfaces demand.
- **Schema migration from v1 to v2.** Out of scope until v2 is needed. The version-bump rules in `docs/export-schema.md` define the contract.
- **`post-commit` git hook integration** that auto-runs `sift accept --by-commit`. Explicit non-goal — preserves "git is read-only for sift" guiding principle.
